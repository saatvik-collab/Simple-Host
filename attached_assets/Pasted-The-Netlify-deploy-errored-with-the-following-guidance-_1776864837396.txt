The Netlify deploy errored, with the following guidance provided:

Diagnosis
- Relevant log lines: the build stops because pnpm refuses to do a frozen install: [line 16](#L16), [line 18](#L18), [line 21](#L21) and the suggestion to update the lockfile at [line 22](#L22). The build ultimately fails at [line 24](#L24). There is also a pre-build warning that NODE_ENV is set to production which will skip devDependencies ([line 15](#L15)).
- Error type and cause: pnpm exits with ERR_PNPM_LOCKFILE_CONFIG_MISMATCH during a frozen install because the current package/overrides configuration in package.json (or workspace config) does not match the checked-in lockfile. Netlify runs a frozen install by default, so pnpm fails instead of updating the lockfile.

Solution
1) Preferred fix — update and commit the lockfile:
- Locally, at the repository root, run:
```bash
git pull
pnpm install --no-frozen-lockfile
git add pnpm-lock.yaml
git commit -m "chore: update pnpm lockfile"
git push
```
- Push will let Netlify run a normal install against the updated lockfile and succeed.

2) Alternative (if you cannot update the lockfile immediately):
- Set an environment variable in Netlify to disable frozen-lockfile for the build: in your Site > Site settings > Build & deploy > Environment > Add variable:
  - Key: PNPM_CONFIG_FROZEN_LOCKFILE
  - Value: false
This tells pnpm to allow updating the lockfile during CI. (Better to update and commit the lockfile long-term.)

3) Address the NODE_ENV warning if your build needs devDependencies:
- If your build step requires devDependencies, unset or change NODE_ENV so devDependencies are installed, or add the specific dev tools as production dependencies. To change Node env on Netlify, go to Site settings > Build & deploy > Environment and remove or change NODE_ENV. Note the build log warning: [line 15](#L15).

Why this fixes it
- pnpm’s frozen install refuses to modify the lockfile; updating the lockfile (or permitting it to update) resolves the mismatch reported at [line 21](#L21) and suggested at [line 22](#L22). Committing the updated pnpm-lock.yaml ensures consistent installs in CI.
